Exécution Docker Serverless

Avancé ⏱ ~75 min

Apprenez à configurer les applications en mode d'exécution serverless, où les conteneurs sont automatiquement démarrés à la demande et arrêtés lorsqu'ils sont inactifs pour optimiser l'utilisation des ressources.

Prérequis

  • Terminé : Démarrage des applications
  • Terminé : Arrêt des applications
  • Compréhension des concepts de réseau Docker et de reverse proxy

Ce que vous apprendrez

  • Comprendre le modèle d'exécution serverless
  • Configurer les paramètres de timeout d'inactivité et d'auto-scaling
  • Mettre en place l'activation de conteneurs basée sur les requêtes
  • Surveiller les performances de cold-start
  • Optimiser les images conteneur pour un démarrage rapide

Fonctionnement du mode Serverless

En mode serverless, le reverse proxy de la plateforme intercepte les requêtes entrantes. Si le conteneur de l'application cible n'est pas en cours d'exécution, il est démarré à la volée. Après une période d'inactivité configurable sans requêtes, le conteneur est arrêté pour libérer les ressources.

Configuration

Activez le mode serverless dans la configuration de votre application :

# Dans le deploy.ini de l'application ou ses métadonnées
execution_mode = serverless
idle_timeout = 300  # secondes avant arrêt automatique
max_cold_start = 30  # temps de démarrage maximum autorisé en secondes
