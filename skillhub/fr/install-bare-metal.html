Installation sur Ubuntu Bare-Metal

Débutant ⏱ ~120 min

Apprenez à déployer la plateforme AI-Powered-Store sur un serveur Ubuntu bare-metal vierge (22.04 ou 24.04 LTS). Ce guide couvre l'ensemble du processus d'installation, de la préparation du système au déploiement final.

Prérequis

  • Une installation Ubuntu 22.04 ou 24.04 LTS propre
  • Accès root ou sudo
  • Au moins 4 Go de RAM et 20 Go d'espace disque
  • Connectivité Internet pour le téléchargement des paquets
  • Un GPU NVIDIA compatible (optionnel, pour le support MIG)

Étape 1 : Installation des dépendances système

Commencez par mettre à jour les paquets et installer les dépendances nécessaires :

sudo apt update
sudo apt --fix-broken install -y
sudo apt install -y python3 python3-pip python3-venv net-tools unzip

Étape 2 : Installation des outils CLI

Installez les outils en ligne de commande nécessaires à la gestion de la plateforme :

Amazon Kiro CLI

curl -fsSL https://cli.kiro.dev/install | bash

OVH CLI (shai)

curl -fsSL https://raw.githubusercontent.com/ovh/shai/main/install.sh | sh
echo 'export PATH="~/.local/bin:$PATH"' >> ~/.bashrc

AWS CLI

curl -s "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip -q awscliv2.zip
sudo ./aws/install
rm -rf aws awscliv2.zip

Terraform

curl -s "https://releases.hashicorp.com/terraform/1.14.5/terraform_1.14.5_linux_amd64.zip" -o "terraform.zip"
unzip -q terraform.zip
sudo mv terraform /usr/local/bin/
rm -f terraform.zip

Étape 3 : Configuration réseau (Netplan)

Le script configure automatiquement les priorités des interfaces réseau pour assurer que l'interface publique est préférée pour le routage par défaut :

  • Interface publique : métrique de route 50 (priorité haute)
  • Interface privée : métrique de route 200 (priorité basse)

La détection des interfaces se fait automatiquement en analysant les adresses IP (plages privées : 10.x, 172.16-31.x, 192.168.x).

Attention : Une sauvegarde du fichier netplan existant est créée automatiquement avant toute modification.

Étape 4 : Installation de Docker

Docker est installé via le script officiel, avec docker-compose en complément :

curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
sudo curl -sL "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
rm -f get-docker.sh
Note : Vous devrez vous déconnecter et reconnecter pour que l'ajout au groupe docker prenne effet.

Étape 5 : Installation des pilotes NVIDIA et support GPU (optionnel)

Si votre serveur dispose d'un GPU NVIDIA compatible :

# Installation des pilotes NVIDIA
sudo apt install -y nvidia-driver-550 nvidia-utils-550

# Activation du mode MIG (Multi-Instance GPU)
sudo nvidia-smi -mig 1

# Installation du NVIDIA Container Toolkit
sudo apt install -y nvidia-container-toolkit

# Vérification de l'accès Docker au GPU
docker run --rm --gpus '"device=0"' nvidia/cuda:12.3.0-base-ubuntu22.04 nvidia-smi
Note : Le mode MIG n'est disponible que sur les GPU NVIDIA A100, A30, H100, et similaires. Cette étape sera ignorée si le matériel n'est pas compatible.

Étape 6 : Configuration AWS (stockage S3 OVH)

Configurez les identifiants AWS pour la synchronisation S3 avec OVH Object Storage :

mkdir -p ~/.aws

# Fichier ~/.aws/config
cat > ~/.aws/config <<EOF
[profile OVH-SWAUTOMORPH]
region = gra
output = json
EOF

# Fichier ~/.aws/credentials
cat > ~/.aws/credentials <<EOF
[OVH-SWAUTOMORPH]
aws_access_key_id = VOTRE_ACCESS_KEY
aws_secret_access_key = VOTRE_SECRET_KEY
endpoint_url = https://s3.gra.io.cloud.ovh.net/
signature_version = s3v4
EOF

# Variables d'environnement
export AWS_DEFAULT_PROFILE=OVH-SWAUTOMORPH
export AWS_ENDPOINT_URL_S3=https://s3.gra.io.cloud.ovh.net/

Étape 7 : Clonage du dépôt

Clonez le dépôt principal avec ses sous-modules :

git clone https://github.com/Sam9682/opcp-ai-powered-store.git
cd opcp-ai-powered-store
git submodule update --init --recursive

Étape 8 : Configuration de l'environnement Python

Créez un environnement virtuel et installez les dépendances :

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

Étape 9 : Configuration finale

Préparez l'arborescence de déploiement :

mkdir -p logs
chmod +x setup_modsecurity_config.sh
cd ~
mkdir -p deployments/admin

Actions post-installation

Important — À ne pas oublier :
  • Modifier ./conf/deploy.ini avec vos paramètres de plateforme, notamment les valeurs PLTF_NAME et DOMAIN
  • Ajouter le certificat SSL dans ~/opcp-ai-powered-store/ssl/fullchain_domain.crt pour le HTTPS nginx
  • Ajouter la clé privée SSL dans ~/opcp-ai-powered-store/ssl/privateKey_domain.key pour le HTTPS nginx
  • Renseigner aws_access_key_id et aws_secret_access_key dans ~/.aws/credentials pour la synchronisation S3

Script automatisé

L'ensemble de ces étapes peut être exécuté automatiquement via le script init_pltf.sh :

chmod +x init_pltf.sh
./init_pltf.sh
