Sauvegardes

Intermédiaire ⏱ ~60 min

Apprenez à sauvegarder les données applicatives, les bases de données et la configuration de la plateforme vers le stockage objet S3 compatible OVH via l'AWS CLI.

Prérequis

  • Terminé : Installation sur Ubuntu Bare-Metal
  • AWS CLI configuré avec les identifiants S3 OVH
  • Instance AI-Powered-Store en fonctionnement

Ce que vous apprendrez

  • Sauvegarde des bases de données applicatives
  • Synchronisation des sauvegardes vers le stockage objet OVH S3
  • Restauration à partir de sauvegardes
  • Mise en place de sauvegardes automatiques planifiées
  • Vérification de l'intégrité des sauvegardes

Sauvegarde de base de données

Créez un dump de la base de données pour une application en cours d'exécution :

# Sauvegarde d'une base PostgreSQL
docker exec <conteneur_bdd> pg_dump -U postgres <base_de_donnees> > sauvegarde_$(date +%Y%m%d).sql

Envoi vers S3 (OVH Object Storage)

# Synchroniser les sauvegardes vers S3
aws s3 sync ./backups/ s3://<nom_bucket>/backups/ \
  --profile OVH-SWAUTOMORPH \
  --endpoint-url https://s3.gra.io.cloud.ovh.net/

Restauration à partir d'une sauvegarde

# Télécharger la sauvegarde depuis S3
aws s3 cp s3://<nom_bucket>/backups/sauvegarde_20240101.sql ./restauration.sql \
  --profile OVH-SWAUTOMORPH \
  --endpoint-url https://s3.gra.io.cloud.ovh.net/

# Restaurer la base de données
docker exec -i <conteneur_bdd> psql -U postgres <base_de_donnees> < restauration.sql
