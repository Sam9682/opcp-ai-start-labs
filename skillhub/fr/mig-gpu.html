Docker avec GPU MIG

Avancé ⏱ ~90 min

Apprenez à utiliser la technologie NVIDIA Multi-Instance GPU (MIG) pour partitionner un seul GPU en plusieurs instances isolées, permettant à plusieurs conteneurs Docker de partager les ressources GPU de manière sécurisée.

Prérequis

  • Terminé : Installation sur Ubuntu Bare-Metal
  • GPU NVIDIA A100, A30, H100 ou compatible avec support MIG
  • Pilotes NVIDIA 550+ et Container Toolkit installés

Ce que vous apprendrez

  • Comprendre l'architecture MIG et le partitionnement GPU
  • Activer et configurer le mode MIG
  • Créer des instances GPU et des instances de calcul
  • Assigner des slices MIG aux conteneurs Docker
  • Surveiller l'utilisation GPU par instance

Activation du mode MIG

# Activer le mode MIG sur le GPU 0
sudo nvidia-smi -i 0 -mig 1

# Vérifier que le mode MIG est activé
nvidia-smi --query-gpu=mig.mode.current --format=csv

Création d'instances GPU

# Lister les profils MIG disponibles
nvidia-smi mig -lgip

# Créer une instance GPU (ex: profil 1g.5gb)
sudo nvidia-smi mig -i 0 -cgi 1g.5gb -C

Exécution Docker avec MIG

# Exécuter un conteneur sur un device MIG spécifique
docker run --rm --gpus '"device=0:0"' nvidia/cuda:12.3.0-base-ubuntu22.04 nvidia-smi
