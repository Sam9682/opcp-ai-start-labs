Arrêt des applications

Intermédiaire ⏱ ~30 min

Apprenez à arrêter proprement les applications, comprendre le processus d'arrêt et gérer les ressources après l'arrêt des conteneurs.

Prérequis

  • Terminé : Démarrage des applications
  • Au moins une application en cours d'exécution

Ce que vous apprendrez

  • Procédures d'arrêt gracieux des applications
  • Arrêt d'applications individuelles vs. toute la plateforme
  • Nettoyage des ressources Docker (volumes, réseaux)
  • Compréhension des hooks d'arrêt et persistance des données

Arrêter une application

Utilisez docker-compose pour arrêter proprement les conteneurs :

# Arrêter une application spécifique
cd ~/deployments/<nom_app>
docker-compose down

# Arrêter avec suppression des volumes (supprime les données)
docker-compose down -v

Arrêter toute la plateforme

# Naviguer vers le répertoire de la plateforme
cd ~/opcp-ai-powered-store

# Arrêter tous les services de la plateforme
docker-compose down
Attention : L'utilisation du flag -v supprimera les volumes et données associés. Créez toujours une sauvegarde avant de procéder.
Labo disponible : Un module de laboratoire pratique accompagne cette leçon. Lancez le labo pour vous exercer dans un environnement isolé.
