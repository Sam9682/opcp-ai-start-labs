Making Backups

Intermediate ⏱ ~60 min

Learn how to back up application data, databases, and platform configuration to OVH S3-compatible object storage using the AWS CLI.

Prerequisites

  • Completed: Installation on Bare-Metal Ubuntu
  • AWS CLI configured with OVH S3 credentials
  • Running AI-Powered-Store instance

What You Will Learn

  • Backing up application databases
  • Syncing backups to OVH S3 object storage
  • Restoring from backups
  • Setting up automated backup schedules
  • Verifying backup integrity

Database Backup

Create a database dump for a running application:

# Backup a PostgreSQL database
docker exec <db_container> pg_dump -U postgres <database> > backup_$(date +%Y%m%d).sql

Upload to S3 (OVH Object Storage)

# Sync backups to S3
aws s3 sync ./backups/ s3://<bucket_name>/backups/ \
  --profile OVH-SWAUTOMORPH \
  --endpoint-url https://s3.gra.io.cloud.ovh.net/

Restore from Backup

# Download backup from S3
aws s3 cp s3://<bucket_name>/backups/backup_20240101.sql ./restore.sql \
  --profile OVH-SWAUTOMORPH \
  --endpoint-url https://s3.gra.io.cloud.ovh.net/

# Restore database
docker exec -i <db_container> psql -U postgres <database> < restore.sql
