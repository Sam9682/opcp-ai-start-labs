Docker with MIG GPU

Advanced ⏱ ~90 min

Learn how to use NVIDIA Multi-Instance GPU (MIG) technology to partition a single GPU into multiple isolated instances, allowing multiple Docker containers to share GPU resources securely.

Prerequisites

  • Completed: Installation on Bare-Metal Ubuntu
  • NVIDIA A100, A30, H100, or compatible GPU with MIG support
  • NVIDIA driver 550+ and Container Toolkit installed

What You Will Learn

  • Understanding MIG architecture and GPU partitioning
  • Enabling and configuring MIG mode
  • Creating GPU instances and compute instances
  • Assigning MIG slices to Docker containers
  • Monitoring GPU usage per instance

Enabling MIG Mode

# Enable MIG mode on GPU 0
sudo nvidia-smi -i 0 -mig 1

# Verify MIG mode is enabled
nvidia-smi --query-gpu=mig.mode.current --format=csv

Creating GPU Instances

# List available MIG profiles
nvidia-smi mig -lgip

# Create a GPU instance (e.g., 1g.5gb profile)
sudo nvidia-smi mig -i 0 -cgi 1g.5gb -C

Running Docker with MIG

# Run a container on a specific MIG device
docker run --rm --gpus '"device=0:0"' nvidia/cuda:12.3.0-base-ubuntu22.04 nvidia-smi
