Serverless Docker Execution

Advanced ⏱ ~75 min

Learn how to configure applications for serverless execution mode, where containers are automatically started on-demand and stopped when idle to optimize resource usage.

Prerequisites

  • Completed: Starting Applications
  • Completed: Stopping Applications
  • Understanding of Docker networking and reverse proxy concepts

What You Will Learn

  • Understanding the serverless execution model
  • Configuring idle timeout and auto-scaling parameters
  • Setting up request-based container activation
  • Monitoring cold-start performance
  • Optimizing container images for fast startup

How Serverless Mode Works

In serverless mode, the platform reverse proxy intercepts incoming requests. If the target application container is not running, it is started on-the-fly. After a configurable idle period with no requests, the container is stopped to free resources.

Configuration

Enable serverless mode in your application configuration:

# In the application deploy.ini or metadata
execution_mode = serverless
idle_timeout = 300  # seconds before auto-stop
max_cold_start = 30  # maximum allowed startup time in seconds
